# AI 코딩 도구 활용 옥외광고 민원 판정 보조 시스템 개발 사례

---

## 사업개요

**❍ 추진기간 :** 2026. 3. 22. ~ 2026. 3. 29. (8일)

**❍ 소요예산 :** 비예산

**❍ 주요내용**
- 강남구 옥외광고 민원 담당자를 위한 내부형 AI 보조 판정 웹앱(광고판정·AdJudge) 개발
- 광고물 설치 조건 입력 시 허가·신고·불가 판정, 법령 근거 조문, 수수료, 구비서류를 1초 이내 자동 산출
- 인터넷 차단 강남구 내부망 환경에서 완전 오프라인 운용 가능하도록 Docker Compose 기반으로 구성

---

## 주요개선사항

| 개선전 | 개선후 |
|--------|--------|
| 담당자가 민원 접수 시마다 옥외광고물법·서울시 조례·강남구 조례를 수동 검색하여 응대 | 광고물 조건(유형·면적·층수·조명·용도지역 등) 입력 → 1초 내 허가/신고/불가 판정 자동 산출 |
| 담당자별 법령 해석 편차로 민원 응답 비표준화 | 동일 입력 → 동일 결과 100% 보장 (규칙 엔진 기반 결정론적 처리) |
| 수수료 계산을 수기로 처리 (오류 발생 가능) | 광고물 유형·면적·조명 종류에 따른 수수료 자동 계산 |
| 법령 근거 조문을 응답에 별도 첨부 불가 | 판정 결과와 함께 해당 법령 조문 자동 제시 |
| 규칙 변경 시 개발자 코드 수정 필요 | 관리자 UI에서 비개발자도 규칙 CRUD 직접 수행 가능 |
| 시스템 개발에 수개월·외부 용역 예산 소요 | AI 코딩 도구(Claude Code) 활용으로 비예산, 8일 내 개발 완료 |

## 세부추진실적

**❍ 기획 및 설계 (3.22)**
- 옥외광고물법·서울시 조례·강남구 심의기준 분석, 판정 대상 광고물 13종 확정
- 규칙 엔진(허가/신고/불가 결정)과 AI 조문 검색(설명·근거 제시)의 역할 분리 아키텍처 설계
- 내부망 완전 오프라인 운용을 위한 기술 스택 확정 (FastAPI·Next.js·PostgreSQL·Docker Compose·Ollama)

**❍ 핵심 시스템 구현 (3.25~3.26)**
- PostgreSQL 데이터베이스 스키마 구축 (판정 규칙·수수료·서류·법령벡터·판정이력 테이블)
- 강남구 심의기준 PDF 기반 판정 규칙 20건 DB 적재
- 규칙 엔진 개발 : 절대 불가 조건(테헤란로·용도지역) → 규격 초과 → 허가/신고 분기 → 심의 분류의 4단계 판정 로직
- 면적 자동 산출 기능 추가 (가로×세로 입력 시 ㎡ 자동 계산)
- 내부망 배포 최적화 : 외부 AI 서비스(Ollama RAG) 제거, 독립 실행 환경 구성

**❍ 규칙 정밀화 및 UI 개선 (3.27~3.29)**
- 판정 정확도 검증 및 규칙 수정 8회 반복 — 매칭률 84% → 98% 달성
- 돌출간판 업종별 예외 규정 처리 방식 개선 (이·미용업, 의료기관·약국 완화기준 반영)
- 모바일 반응형 UI 전면 재설계 — 현장(민원창구)에서 태블릿·스마트폰 활용 가능
- 관리자 패널 구현 : 규칙 CRUD, 판정 이력 로그 조회

## 기대효과

**❍** 민원 응대 시간 단축 및 담당자 법령 검색 부담 경감 — 1건 처리 시간 약 10~15분 → 1분 이내 단축 예상

**❍** 동일 입력 → 동일 결과 보장으로 민원 응답 표준화, 행정 신뢰성 및 민원인 만족도 향상
