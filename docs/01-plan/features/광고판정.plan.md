## 1. 제품 개요

### 1.1 제품명
광고판정 (AdJudge)

### 1.2 목적
강남구 옥외광고 민원 응대 담당자를 위한 내부형 AI 보조 웹앱.
광고물 설치 조건 입력 시 허가·신고·불가 판정과 근거 조문, 행정 절차, 수수료를 일관된 형식으로 제공.

### 1.3 서비스 포지션
- 시작: 참고용 실무 자문 보조도구
- 목표: 준공식 자문도구 수준
- 원칙: 최종 판단은 담당자 수행

---

## 2. 핵심 설계 원칙

1. **판정은 규칙 엔진이 수행한다** — LLM이 판정하지 않음
2. **설명은 조문 기반으로 생성한다** — LlamaIndex RAG가 근거 조문 검색
3. **모든 결과에는 출처와 버전이 포함된다**
4. **동일 입력 → 동일 결과 보장** — 확률적 응답 없음
5. **법령은 계층 구조로 처리한다** — 국가법령 > 서울시 조례 > 강남구 조례
6. **데이터 업데이트는 관리자 UI에서 수행**

## 4. 기술 스택

### 4.1 확정 스택

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | **Next.js (App Router)** | SSR, 서버컴포넌트로 내부망 최적화 |
| Backend | **Python FastAPI** | LlamaIndex 생태계, 임베딩 처리 강점 |
| Database | **PostgreSQL + pgvector** | 관계형 + 벡터 검색 통합 |
| RAG 프레임워크 | **LlamaIndex** | 문서 특화 RAG, 코드량 최소화 |
| LLM (로컬) | **Ollama** | 내부망 완전 오프라인, 자체 호스팅 |
| 임베딩 모델 | **nomic-embed-text** (Ollama) | 한국어 지원, 로컬 실행 |

### 4.2 배포 환경
- **강남구 내부망** (인터넷 차단)
- Docker Compose로 전체 서비스 구성
- 모든 의존성(모델 파일, pip 패키지) 사전 다운로드 후 내부 이전 필요

### 4.3 패키지
```bash
# Backend
pip install llama-index-core
pip install llama-index-vector-stores-postgres
pip install llama-index-llms-ollama
pip install llama-index-embeddings-ollama
pip install fastapi uvicorn

# Frontend
npx create-next-app@latest
```

## 5. 아키텍처: 규칙 엔진 + RAG 완전 분리

```
[사용자 입력 폼]
        ↓
┌───────────────────────────────────┐
│           FastAPI Backend          │
│                                   │
│  ┌─────────────────┐              │
│  │   규칙 엔진      │  ← PostgreSQL │
│  │ (허가/신고/불가)  │    (rule_*   │
│  │  수수료 계산     │     테이블)   │
│  └────────┬────────┘              │
│           │ 판정 결과              │
│  ┌────────▼────────┐              │
│  │  LlamaIndex RAG │  ← pgvector  │
│  │  (근거 조문 검색) │   (임베딩)   │
│  │  (챗봇 답변)     │              │
│  └─────────────────┘              │
└───────────────────────────────────┘
        ↓
[결과 출력: 판정 + 조문 근거 + 수수료 + 서류]
```

**역할 분리 원칙:**
- **규칙 엔진**: 허가/신고/불가 결정, 심의 분류, 수수료 계산 → 100% 결정론적
- **LlamaIndex RAG**: 조문 근거 텍스트 검색, 담당자 추가 질문 답변 → 설명 역할만

## 6. 기능 요구사항 (구체화)

### FR-1: 입력 폼 (구조화 폼 우선)

**필수 입력 항목:**

| 항목 | 타입 | 값 예시 |
|------|------|---------|
| 광고물 유형 | Select | 돌출간판, 벽면이용간판 등 13종 |
| 설치 층수 | Number | 1, 2, 3... |
| 면적 (㎡) | Number | 5.5 |
| 조명 여부 | Radio | 없음 / 내부조명 / 네온·전광류·디지털 |
| 용도지역 | Select | 일반상업 / 준주거 / 일반주거 / 전용주거 등 |
| 자사광고 여부 | Radio | 자사 / 타사 |
| 도로 접면 여부 | Checkbox | 앞면도로 / 뒷면도로 / 테헤란로 접면 |

**추가 질문 (유형별 동적 표시):**
- 돌출간판: 테헤란로 접면 여부 → 설치 불가 조건
- 지주이용간판: 업체 수 (5개 이상 조건)
- 입간판: 보도 유무

### FR-2: 판정 엔진

**판정 흐름:**
```
1. 절대 불가 조건 체크 (최우선)
   - 테헤란로 접면 + 돌출간판 → 불가
   - 용도지역 금지 유형 → 불가

2. 규격 초과 체크
   - 면적, 높이, 돌출폭 기준 초과 → 불가 또는 심의 필요

3. 허가 vs 신고 분기
   - 전광류·디지털·네온 → 허가
   - 일반 간판 규격 이내 → 신고

4. 심의 분류
   - 본심의 / 소심의 / 서울시심의
```

### FR-3: 결과 출력 포맷

```json
{
  "판정": "신고",
  "심의": "소심의",
  "근거조문": [
    { "법령": "옥외광고물법 제3조 제1항", "내용": "...", "버전": "2024.01.01" }
  ],
  "최대규격": { "면적": "5㎡ 이하", "세로": "80cm 이내" },
  "수수료": { "기본": 20000, "조명가중치": 1.5, "합계": 30000 },
  "표시기간": "3년",
  "서류": ["신고서", "도면", "건물사용승낙서", "원색사진"],
  "주의사항": ["테헤란로 접면 시 설치 불가"]
}
```

### FR-4: 수수료 계산

**계산 공식 (PDF 기준):**
```
기본 수수료 = 유형별 기본가 + (면적 - 기준면적) × 추가단가
최종 수수료 = 기본 수수료 × 조명가중치

조명가중치:
- 일반: × 1.0
- 내부조명(형광등, LED): × 1.5
- 네온류·전광류·디지털: × 2.0
```

### FR-5: LlamaIndex RAG (조문 근거 + 챗봇)

**법령 데이터 소스:**
- 국가법령정보공단 Open API (사전 수집 후 내부 저장)
- 강남구 조례 PDF (청크 분할 후 임베딩)
- 옥외광고물 설치가이드 PDF

**역할:**
- 판정 근거 조문 원문 검색 및 표시
- 담당자 추가 질문 답변 (챗봇)
- "이 경우 예외가 있나요?" 등 자연어 질문 처리

### FR-6: 관리자 기능

- 규칙 테이블 CRUD (비개발자 수정 가능 UI)
- 법령 버전 관리 + 히스토리
- 판정 로그 조회
