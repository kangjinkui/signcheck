# 작업 세션 요약 — 2026-03-22

## 개요

`/api/v1/judge` 엔드포인트 테스트 중 `max_spec` 필드가 전부 `null`로 반환되는 문제를 발견하고,
법령 원문을 분석해 `rule_condition` / `rule_effect` 테이블에 전체 간판 유형 규칙을 삽입한 작업.

---

## 1. 문제 발견

### 테스트 요청
```json
{
  "sign_type": "돌출간판",
  "floor": 3,
  "area": 4.5,
  "light_type": "internal",
  "zone": "일반상업지역",
  "ad_type": "self",
  "tehranro": false
}
```

### 문제 증상
- `decision`: `"report"` — 잘못된 판정 (돌출간판은 시행령 제4조 **허가** 대상)
- `max_spec`: 모든 값 `null` — 규격 정보 없음

### 근본 원인
`rule_condition`, `rule_effect` 테이블이 **완전히 비어있음** (0건)

규칙이 없으므로 엔진이 항상 기본값 fallback:
```python
# engine/rule_engine.py:82
return JudgeResult(
    decision="report",
    review_type="소심의",
    display_period="3년",
    warnings=["해당 조건의 상세 규칙을 담당자가 직접 확인하세요."]
)
```

```sql
-- 조회 쿼리
SELECT rc.sign_type, rc.floor_min, rc.floor_max, rc.zone, rc.ad_type, rc.priority,
       re.decision, re.review_type, re.max_area, re.max_height, re.max_width,
       re.max_protrusion, re.display_period
FROM rule_condition rc
JOIN rule_effect re ON rc.id = re.rule_id
ORDER BY rc.sign_type, rc.priority;
```
